# 1.6.0

_Current version. Entries below are populated as tasks complete; final list compiled at release._

## Features
- Export now resolves all library placeholders inside `CMakePresets.json`. The profiling presets (`x64-Debug-Profile`, `x64-Release-Profile`) ship with the correct `<LIB_SHORT>_PROFILING` cache variable instead of the literal template token, so newly created libraries support profile builds without manual edits.
- Upgrade now preserves developer-added macro definitions in `CMakePresets.json` and `CMakeSettings.json`. Custom `cacheVariables` keys (presets), custom `-D…` tokens in `cmakeCommandArgs`, and custom `variables[]` entries (settings) survive a template refresh. Template-managed values still take precedence on conflicting keys; quoted values like `-DPATH="C:/Some Dir"` round-trip intact.

## Bugfixes
- Doxygen `// @file LibraryName*.h` comments at the top of every generated header are now substituted with the project's actual library name on export.

## Deprecations
- Removed `CMakeSettings.json` handling. The file has been dropped from the QT_cmake_library_template (template version 1.7.0); CMakePresets.json — the portable, IDE-agnostic equivalent — is now the only build-config file the tool manages. Existing user projects keep their `CMakeSettings.json` untouched on upgrade; the tool simply ignores it.
